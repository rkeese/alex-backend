Here is a summary of the changes:

4.  **Route Protection**: Updated index.ts with a Global Navigation Guard.
    *   It checks `authStore.mustChangePassword`.
    *   If `true`, it redirects the user to `/change-password` regardless of where they try to navigate (unless the route is the change password page itself).
    *   Exceptions are made for public routes (Login, Register).

### Usage Flow
1.  User logs in.
2.  If the token contains `must_change_password: true`, the router immediately redirects them to `/change-password`.
3.  User enters a new password.
4.  System updates the password and clears the flag via API.
5.  User is logged out and asked to log in again with the new credentials.

You can now test this by setting the `must_change_password` flag to `true` for a user in your backend database and attempting to log in.
